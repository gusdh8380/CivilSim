# CivilSim Mini SimCity 기획서 (확정안 v1.1)

기준일: 2026-03-03  
목적: 다음 개발 단계(코드 작업) 전에 기획 범위와 밸런스 방향을 고정한다.

## 1. 제품 방향

- 장르: 미니 시티빌더 (심시티 감성, 경량 시뮬레이션)
- 핵심 재미:
  1. 플레이어가 도로, 지반, 건물을 직접 배치해 도시를 성장시킨다.
  2. 배치 결정이 월별 수치(예산, 인구, 수요)로 즉시 피드백된다.
  3. 운영 선택(정책, 공공, 유틸리티)이 장기 효율 차이를 만든다.

## 2. 절대 원칙

- 자동 개발 금지:
  - 구역(R/C/I)은 정보 레이어이며 자동 건설 트리거로 사용하지 않는다.
  - 모든 건물은 플레이어 수동 배치로만 생성된다.
- v1.1 인프라는 수치 기반 운영으로 시작한다:
  - 전선, 파이프 시각 네트워크는 후속 단계.
- 차량 에이전트 교통 시뮬레이션은 제외한다.

## 3. 핵심 플레이 루프

1. 지반 정비(G)
2. 도로 배치(F)
3. 구역 지정(Z, R/C/I/X)
4. 건물 수동 배치(B)
5. 시간 진행(월 정산)
6. 정책 조정, 철거, 재배치

## 4. 건물 카테고리 역할과 당위성

### 4.1 주거(Residential)

- 역할: 인구 공급원
- 당위성: 상업과 공업의 수익/고용 효율을 올리는 기반
- 핵심 지표: ResidentCapacity

### 4.2 상업(Commercial)

- 역할: 주민 기반 세수 엔진
- 당위성: 주민 소비를 수익으로 전환
- 핵심 지표: JobCapacity, 월 유지비 대비 세수 효율

### 4.3 공업(Industrial)

- 역할: 대량 고용 엔진
- 당위성: 인구 성장 이후 고용 수요를 빠르게 흡수
- 핵심 지표: JobCapacity, 유지비/유틸 부담

### 4.4 공공(Public)

- 역할: 운영 안정화와 보너스 제공
- 당위성: 직접 수익은 낮아도 전체 운영 배율을 개선
- 핵심 지표: ServiceValue(교육, 안전, 의료 등)

### 4.5 유틸리티(Utility)

- 역할: 전기/수도 공급
- 당위성: 공급 부족 시 도시 전체 운영률 하락
- 핵심 지표: PowerSupply, WaterSupply

### 4.6 인프라(Infrastructure)

- 역할: 접근성 기반
- 당위성: 도로 미연결 상태는 운영 패널티 대상
- 상태: 카테고리 정의만 있고 전용 건물 데이터는 후속 추가

## 5. 시스템 규칙

### 5.1 배치 규칙

- 건물은 Foundation 셀에만 배치 가능
- 도로 인접은 배치 조건이 아니라 운영 효율 조건으로 반영
- 자금 부족 시 배치 실패

### 5.2 경제 규칙

- 기본 수입:
  - `(주거 인원 * 주거세) + (고용 인원 * 고용세)`
- 기본 지출:
  - `모든 건물의 MaintenanceCostPerMonth 합`
- 수요 배율:
  - 기존 Demand 점수 기반 배율 유지
- 게임 결과:
  - 파산 및 연속 적자 조건 기반 패배 로직 유지

### 5.3 인구/수요 규칙

- 인구:
  - 배치된 건물 ResidentCapacity 합
- 수요:
  - 주거 수요: `고용 - 거주`
  - 상업/공업 수요: `거주 기반 파생값 - 현재 고용량`
- 수요는 우선 수치 피드백 중심으로 운영

## 6. 전기/수도 인프라 MVP (확정)

### 6.1 설계 방식

- 단계 1:
  - 도시 전체 공급/수요 풀 방식
  - 건물별 경로 탐색 없음
- 단계 2(후속):
  - 전선/파이프 네트워크 시각 및 연결 판정 추가

### 6.2 월말 계산식

- `PowerRate = clamp(총전력공급 / 총전력수요, 0~1)`
- `WaterRate = clamp(총수도공급 / 총수도수요, 0~1)`
- `OperationRate = min(PowerRate, WaterRate)`

### 6.3 운영 반영

- 주거 유효 인구:
  - `ResidentCapacity * OperationRate`
- 비주거 유효 고용:
  - `JobCapacity * OperationRate`
- 공공 서비스 효과:
  - `ServiceValue * OperationRate`

### 6.4 HUD/리포트 노출

- 월간 보고에 다음 항목 추가:
  - 전력 공급률, 수도 공급률, 최종 운영률
  - 부족 원인(전력 부족, 수도 부족, 복합 부족)

## 7. 밸런스 가이드라인 (1차 확정)

### 7.1 경제 전역 기본값(목표)

- InitialBudget: `80,000`
- TaxPerResidentPerMonth: `35`
- TaxPerJobPerMonth: `30`
- BankruptcyThreshold: `-20,000`

### 7.2 카테고리별 표준 체급

- 주거 1x1:
  - 건설 1,500 / 유지 45 / 거주 4
- 주거 2x1:
  - 건설 3,000 / 유지 90 / 거주 8
- 상업 1x1:
  - 건설 2,800 / 유지 85 / 고용 5
- 상업 2x1:
  - 건설 5,600 / 유지 170 / 고용 10
- 공업 1x1:
  - 건설 3,200 / 유지 120 / 고용 10
- 공공 1x1:
  - 건설 7,000~9,000 / 유지 220~300 / 고용 6~10 + 서비스값
- 유틸 1x1:
  - 변전소: 건설 8,500 / 유지 260 / 전력 공급 220
  - 펌프장: 건설 8,200 / 유지 240 / 수도 공급 200
  - 폐기물: 건설 9,000 / 유지 280 / 서비스 보정

### 7.3 운영 배율 구성

- 최종 수입:
  - `BaseIncome * DemandMultiplier * ServiceMultiplier * OperationRate`
- ServiceMultiplier 범위:
  - `0.85 ~ 1.15` 권장

## 8. UI/입력 스펙

- B: 건물 패널 토글
- F: 도로 모드 토글
- G: 지반 모드 토글
- Z: 구역 모드 토글
- R/C/I/X: 구역 타입 전환/해제
- ESC/RMB: 현재 모드 취소
- 정책/월간보고/환경설정: HUD 버튼으로 패널 열기/닫기

## 9. 콘텐츠 범위 (현재 에셋 기준)

- 현재 건물 데이터 총 25종
  - 주거 6, 상업 9, 공업 4, 공공 3, 유틸 3
- 아이콘 기반 UI 사용
- 사운드는 임시 또는 무음 운영

## 10. 구현 우선순위 (다음 개발 단계)

1. BuildingData 확장:
   - PowerSupply, WaterSupply, ServiceValue, ServiceType
2. UtilityManager 추가:
   - 전력/수도 공급-수요 계산, OperationRate 발행
3. EconomyManager 연동:
   - OperationRate, ServiceMultiplier 반영
4. HUD/월간보고 확장:
   - 전기/수도/서비스 상태 표시
5. 데이터 리밸런싱:
   - 기존 25개 에셋 수치 일괄 조정

## 11. 완료 기준 (Definition of Done)

- 20~30분 플레이에서 다음이 성립해야 한다:
  - 확장 순서 선택(주거, 상업, 공업, 공공, 유틸)에 의미가 있다.
  - 유틸 부족 시 즉시 게임오버가 아니라 운영률 저하로 체감된다.
  - 월간 수치가 재배치/정책 조정으로 개선된다.
  - 패널 중복 오픈, 모드 충돌, 소프트락이 없다.
